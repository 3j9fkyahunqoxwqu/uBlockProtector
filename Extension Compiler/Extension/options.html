Common Questions and Problems

Will there be a configurable white list?

Not until I have a reliable way to synchronously send configuration data to content scripts. This means probably not anytime soon.

uBlock Protector is causing further breakage.

Report it and I will take care of it within 2 days.

Website X is not working.

uBlock Protector is not magic, a lot of websites need to be fixed on a case by case basis. Report it and I will have a look. Depending on the website, it can take some time.

Workarounds

viasatsport.se, viasport.fi, tv3sport.dk, viasport.no

Add the following filters to uBlock Origin. No ads shown, video not blurred, but may break some player functionalities.

@@||fwmrm.net^$domain=viasatsport.se|viasport.fi|tv3sport.dk|viasport.no
||freewheel-mtgx-tv.akamaized.net^$media,domain=viasatsport.se|viasport.fi|tv3sport.dk|viasport.no
||fwmrm.net^$image,important,domain=viasatsport.se|viasport.fi|tv3sport.dk|viasport.no
||fwmrm.net/ad/*$xmlhttprequest,important,redirect=nooptext,domain=viasatsport.se|viasport.fi|tv3sport.dk|viasport.no

For Developers

You need to edit and load the unpacked extension if you want to enable debug mode and/or to change rules.

Debug switch is in common.js. It may be already on when you pull the repository.

Documentation of the API is in content/content-core.js, content/content-domlib.js, and background/background-core.js.

Rules are in content/content-rules.js and background/background-rules.js.

Some rules that will help with debugging, along with rules that are work in progress, are in content/content-debug.js and background/background-debug.js. They will only run in debug mode.
